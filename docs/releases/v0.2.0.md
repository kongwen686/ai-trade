## AI Trade v0.2.0

本版本把系统从“信号预测与研究工作台”升级为带安全护栏的自动量化交易终端。

### Highlights

- 新增 Auto Trade 页面：`/trading`
- 新增自动交易 API：`POST /api/trading/run`
- 新增自动交易 CLI：`trade-signal-autotrade`
- 支持单次执行和循环执行：`--loop --interval-seconds`
- 支持 `paper` 模拟执行模式，默认不会真实下单
- 支持受保护的 `live` 实盘模式
- 接入 Binance Spot 市价买入 / 市价卖出和 `order/test`
- 自动按预测分数、量比、买盘压力筛选交易候选
- 增加最大持仓数、最大总敞口、单笔投入、冷却时间等风控参数
- 增加止损 / 止盈检查和本地持仓状态持久化
- 实盘成交后使用 Binance 返回的 `executedQty` / `cummulativeQuoteQty` 记录实际仓位
- 卖出时按交易对 `LOT_SIZE.stepSize` 向下取整数量
- 运行配置页新增 Auto Trade 参数组
- 交易终端视觉升级：新增品牌化导航、Auto Trade 模块和深色科技终端界面

### Safety

- 默认 `paper` 模式，只写入本地 `data/trading_state.json`
- `live` 模式必须额外设置：

```bash
export AI_TRADE_LIVE_CONFIRM="I_UNDERSTAND_REAL_ORDERS"
```

- 默认开启 `Use Binance order/test`，只校验订单参数，不真实成交
- 只有关闭 `order/test`，并满足实盘确认变量后，才会调用 Binance Spot `POST /api/v3/order`
- 本地交易状态文件 `data/trading_state.json` 已加入 `.gitignore`

### Web Entrypoints

- `/` 实时扫描
- `/backtest` 历史回测
- `/trading` 自动量化
- `/settings` 运行配置
- `/api/scan` 扫描 API
- `/api/backtest` 回测 API
- `/api/trading/status` 自动交易状态 API
- `POST /api/trading/run` 自动交易执行 API
